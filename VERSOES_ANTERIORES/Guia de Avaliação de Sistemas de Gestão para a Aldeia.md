# Guia de Avaliação de Sistemas de Gestão para a Aldeia

A escolha do sistema de gestão é uma das decisões mais críticas para a Aldeia. Um sistema genérico de clínica médica não atende à complexidade de uma clínica de neurodesenvolvimento (TEA/ABA), que exige acompanhamento multidisciplinar, coleta de dados estruturada e faturamento complexo com convênios.

Este guia foi desenhado para você levar para as demonstrações e assumir o controle da negociação, garantindo que o sistema atenda às necessidades reais da Aldeia.

## 1. Funcionalidades Inegociáveis (O que o sistema DEVE ter)

Para uma clínica de neurodesenvolvimento, o sistema precisa ir além da agenda e do financeiro. Ele deve ser o coração da operação clínica e administrativa [1].

### Gestão Clínica e Terapêutica
*   **Prontuário Eletrônico Multidisciplinar:** Psicólogos, fonoaudiólogos e TOs precisam acessar o mesmo prontuário, mas com visões e permissões específicas.
*   **Protocolos ABA Nativos:** O sistema deve suportar a inserção e acompanhamento de protocolos como PEI (Plano de Ensino Individualizado), VB-MAPP, ABLLS-R e Denver [1].
*   **Coleta de Dados e Gráficos de Evolução:** Capacidade de registrar tentativas, acertos e comportamentos por sessão, gerando gráficos automáticos de evolução (essencial para relatórios de convênio e para os pais) [1].

### Gestão Administrativa e Financeira
*   **Agenda Inteligente:** Visualização por profissional, por sala e por paciente. Detecção automática de conflitos (ex: não agendar duas terapias no mesmo horário para a mesma criança) [1].
*   **Faturamento TISS Automatizado:** Geração de guias TISS (padrão exigido pela ANS) em lote, com validação automática para evitar glosas (recusas de pagamento pelos convênios) [2].
*   **Controle de Repasses (Split de Pagamento):** Cálculo automático da comissão dos terapeutas (ex: 35% para a clínica, 65% para o terapeuta) com base nos atendimentos realizados e efetivamente pagos.

### Relacionamento com a Família
*   **Portal ou App para os Pais:** Um ambiente onde a família possa ver a agenda, receber lembretes, visualizar a evolução da criança e acessar orientações para casa [1].

## 2. O Roteiro de Perguntas para a Demonstração

Não deixe o vendedor apenas mostrar as telas bonitas. Faça estas perguntas para testar os limites do sistema [3]:

### Sobre a Operação Clínica (TEA/ABA)
1.  *"Como o sistema lida com a coleta de dados de uma sessão ABA? Ele gera gráficos de evolução automaticamente ou o terapeuta precisa exportar para o Excel?"*
2.  *"É possível personalizar os formulários de anamnese e evolução para as diferentes especialidades (Fono, TO, Psico) dentro do mesmo prontuário do paciente?"*
3.  *"O sistema possui integração nativa com protocolos como VB-MAPP ou Denver, ou teremos que anexar PDFs externos?"*

### Sobre Faturamento e Convênios (Onde mora o risco financeiro)
4.  *"Como funciona o módulo de faturamento TISS? O sistema faz uma pré-validação das guias antes do envio para a operadora para evitar glosas por erro de digitação?"*
5.  *"Se um convênio glosar (negar) um pagamento, como o sistema me ajuda a gerenciar o recurso de glosa?"*
6.  *"O sistema calcula automaticamente o repasse dos terapeutas apenas sobre as sessões que o convênio já pagou, ou ele calcula sobre o que foi faturado?"*

### Sobre Suporte e Segurança
7.  *"Se o sistema cair no meio da tarde, qual é o SLA (tempo de resposta) do suporte técnico? O suporte é por robô, chat ou telefone?"* [3]
8.  *"Como vocês garantem a adequação à LGPD, considerando que lidamos com dados sensíveis de menores de idade com TEA?"* [1]

## 3. Como Diferenciar Qualidade (Os "Red Flags")

Durante a demonstração, preste atenção nestes sinais para separar um sistema excelente de um sistema medíocre:

| Critério de Avaliação | Sinal de Qualidade (O que buscar) | Red Flag (Sinal de Alerta) |
|-----------------------|-----------------------------------|----------------------------|
| **Foco no Nicho** | O vendedor entende termos como "PEI", "ABA" e "Glosa". O sistema tem módulos específicos para autismo. | O vendedor tenta adaptar um sistema de clínica médica geral dizendo que "dá para usar o campo de observação". |
| **Usabilidade (UX)** | A tela é limpa, intuitiva e exige poucos cliques para registrar uma evolução ou agendar uma sessão. | O sistema parece um software dos anos 90, cheio de abas confusas e botões pequenos. |
| **Automação** | O sistema envia lembretes de consulta por WhatsApp automaticamente e gera guias TISS em lote. | Tudo exige intervenção manual (ex: ter que clicar um por um para enviar lembrete). |
| **Relatórios** | Dashboards visuais que mostram taxa de ocupação, inadimplência e evolução clínica em tempo real. | Relatórios que precisam ser exportados para Excel para fazerem sentido. |

## 4. Sistemas Recomendados para Avaliação

Com base na pesquisa de mercado, recomendo que você agende demonstrações com pelo menos duas destas plataformas especializadas:

1.  **BlueSmiles / BlueFamily:** Fortemente focado em clínicas TEA e ABA. Possui protocolos nativos (VB-MAPP, Denver), coleta de dados estruturada e um app específico para as famílias acompanharem a evolução [1].
2.  **ABA Central:** Plataforma dedicada exclusivamente à gestão de terapia ABA, conectando profissionais, clínicas e famílias.
3.  **Neo Aba:** Focado na automação de protocolos (VB-MAPP, ABLLS-R) e otimização do tempo do terapeuta.

**Dica de Ouro:** Peça para o vendedor simular o fluxo completo de um paciente: desde o agendamento da primeira avaliação, passando pelo registro de uma sessão ABA, até a geração da guia TISS para o convênio. Isso revelará a verdadeira fluidez do sistema.
